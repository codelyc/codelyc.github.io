git status三种状态变化

  • 红色：文件没有被管理，使用git add，文件颜色变为绿色
  • 绿色：git已经将文件管理起来了，git commit -m ‘文件描述信息’
  • 文件名消失：文件已经生成了一个版本了

git打标签

  • 打标签的作用，就是给项目的开发节点，加上语义化的名字，也即功能版本的别名。
  • 打上标签名的同时，写上附带信息，可以方便项目日后维护过程中的回溯和复查。
  • 另外，也可以通过标签记录，大致了解当前项目的向下兼容性、API的修改和迭代情况。

一份文集等待出版，有a、b、c、d四篇。现在通过Git管理进度。

给开源项目贡献代码

  1. fork源代码
  2. 在自己的仓库进行修改代码
  3. git clone [网上代码]
  4. git push origin master
  5. 给源代码的作者提交修复bug的申请(pull request) 点击修改项目界面中的pull request
  6. 点击create pull request就能提交 把请求发送过去了

优先级：本地>全局>系统配置文件

  • 系统配置文件：/etc/.gitconfig
