服务端Agent开发

kafka

Kafka是⼀一个分布式数据流平台，可以运⾏在单台服务器器上，也可以在多台服务器器上部署形成集群。它提供了了发布和订阅功能，使⽤用者可以发送数据到Kafka中，也可以从Kafka中读取数据。

[特点]：

Kafka具有⾼高吞吐、低延迟、⾼高容错。

kafka框架

  • Producer：消息的产⽣生者。
  • kafka cluster： kafka集群，一台或多台服务器组成
    • Broker： Broker是指Kafka实例的服务器节点。每个服务器器上有一个或多个kafka的实例，每个kafka集群内的broker都有一个不重复的编号，如图中的broker-0、 broker-1等……
    • Topic：可以理理解为消息的分类， kafka的数据就保存在topic。在每个broker上都可以创建多个topic。通常是一个业务线建一个topic。
    • Partition：把同一个topic分成不同的分区，提高负载
      1. leader:分区的主节点（老大）
      2. follower：分区的从节点（小弟）
    • Consumer：消费者，即消息的消费方，是消息的出口。

工作流程

  1. ⽣产者从Kafka集群获取分区leader信息
  2. ⽣产者将消息发送给leader
  3. leader将消息写入本地磁盘
  4. follower从leader拉取消息数据
  5. follower将消息写⼊本地磁盘后向leader发送ACK
  6. leader收到所有的follower的ACK之后向⽣产者发送ACK

Kafka选择分区的模式

  1. 指定往哪个分区写
  2. 指定key，kafka根据key做hash然后决定写哪个分区
  3. 轮询方式

生产者往kafka发送数据的模式（3种）

  1. 0：把数据发给leader就成功，效率最高、安全性最低。
  2. 1: 把数据发送给leader，等待leader回ACK
  3. all:把数据发给leader,确保follower从leader拉取数据回复ack给leader，leader再回复ACK；安全性最高

最后要注意的是，如果往不存在的topic写数据， kafka会⾃动创建topic， partition和replication的数量默认配置都是1。

部署环境

  1. 下载JDK(12.0.1)

  2. 下载kafka,版本为kafka_2.12-2.3.0.tgz

  3. 将下载好的压缩包解压到本地

  4. 修改zookeeper配置文件（ zookeeper用来维护集群列表的）

  5. 开启zookeeper

  6. 修改kafka配置文件

  7. 启动kafka
