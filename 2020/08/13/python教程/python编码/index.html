Python 编码总结

unicode和utf-8关系

unicode

unicode就是一个字符集，一个字符都有各自唯一的ID号码（一般为16位数字）

utf-8

根据unicode这个字符集，将字符的id号进行压缩编码，便于更好的传输

啥时候用unicode，啥时候用utf-8:？

  • 在计算机内存中，统一使用Unicode编码，当需要保存到硬盘或者需要传输的时候，就转换为UTF-8编码。

  • 用记事本编辑的时候，从文件读取的UTF-8字符被转换为Unicode字符到内存里，编辑完成后，保存的时候再把Unicode转换为UTF-8保存到文件：

  • 浏览网页的时候，服务器会把动态生成的Unicode内容转换为UTF-8再传输到浏览器

encode与decode到底做了啥

python3默认编码是unicode，意思是我的字符串只有按照unicode编码我才能显示出来，即：python3的字符串都是unicode，其他所有的编码（utf-8，gbk）需要转成unicode才能正常显示

由上可知：字符串通过编码转换成字节码（可以理解为每个字符所对应的id号而组成的一串数字），字节码通过解码成为字符串

系统默认编码和本地默认编码区别

系统默认编码

在python3编译器读取.py文件时，若没有头文件编码声明，则默认使用“utf-8”来对.py文件进行解码。（意思是系统默认这个文件是utf-8来编码的，我python解释器为了读取该文件则将其解码为unicode）

本地默认编码

在你编写python3程序时，若使用了 open( )函数 ，而不给它传入 encoding 这个参数，那么会自动使用本地默认编码(操作系统的编码)

解决办法:

如果你的文件保存的是gbk编码，在win 下就不用指定encoding了，或者你自己win默认编码器是utf-8也不需要

注意点

  • Unicode 是一个大的集合包括Gbk

  • window系统默认的是Gbk

  • python编译器默认的是“Unicode”

字符编码和转码

【注意】

  • string数据类型是Unicode，bytes是utf-8。
  • encode(‘utf-8’)编码 string to byte
  • decode(‘utf-8’)解码 bytes to string
