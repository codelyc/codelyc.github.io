python 包导入

参考文章

__name__用法

__name__在本模块中值为__main__,当被其他模块导入的时候，值为文件名，在pycharm输入main然后回车，会自动补全if __name__ == '\__main__'

系统sys.path添加的路径

  • sys.path直接添加了python编译器安装目录下的:lib
  • sys.path自动递进添加了执行文件所在的当前目录与其父目录

import导入包

import基本用法

from 与import搭配使用

  • import后面如果接的是模块名字，则调用模块里面的变量时需要加上模块名当前缀
  • import后面接的是成员，则不需要添加前缀
  • from后面接的是模块名父目录的相对路径，则import后面就写为模块名
  • from后面接的是模块名相对路径，则import后面就写成员

import xxx和from xx import *区别

  • 第一种方式使用模块成员的时候必须加上模块名为前缀
  • 第二种调用模块中成员的时候，直接使用成员名即可，容易产生冲突

使用as起别名

as 后面接别名

模块循环引用

假如两个模块a,b。a import b，并且b import a。相当于组成了一个环形。当代码编译的时候会报错

如上图，

  • 先执行import b,于是去b.py文件中导入代码
  • 到达b.py里面后，因为有import a,所以又跳转到a.py文件导入代码
  • 当执行b.funcb()的时候，因为内存中b.py文件只执行了import a，后面代码还没执行完毕，所以a.py里面的def funcb()是没有导入到b.py中的

【总结】：只要模块导入呈现环状就一定会报错

模块路径添加（sys.path）

  • 调用sys模块，将模块路径添加至sys.path中
  • 调用os模块，搭配os.path.dirname()获得父路径
  • __file__表示当前文件的绝对路径

包和模块的区别

模块就相当于是一个py文件

包则相当于是一个文件夹，文件夹下面有多个模块，并且还包含__init__.py文件，导入包相当于导入该__init__.py文件

直接导入包

导入一个包名不是意味着包下面的所有的内容可以使用，导入一个包实际上是执行了包下面的__init__.py文件

绝对导入

【注意】

在执行一个py脚本的时候,这个脚本以及和这个脚本同级的模块中只能用绝对导入

【缺点】

所有的导入都要从一个根目录下往后解释文件夹之间的关系，如果当前导入包的文件和被导入的包的位置关系发生了变化,那么所有的init文件都要做相应的调整

如上图，因为a.py文件为执行文件，所以ys.path自动递进添加了它的当前目录与其父目录，所以模块内部则根据这些目录进行绝对位置的导入。

例如api文件中的__ini__.py文件内容：F:\code\python已经添加到sys.path了，所以直接利用绝对路径导入./moudle/api

相对路径导入

基本格式

  • .表示当前的路径
  • ..表示的是父路径
  • ...表示父亲的父亲路径

缺点

  • 含有相对导入的py文件不能被直接执行必须放在包中被导入的调用才能正常的使用,例如头文件中包含： form ./xx import a 和from ../xx import a

注意

..或者.为开头，只能在一个包中使用，不能用于不同目录内（form ./xx import a 和from ../xx import a）
