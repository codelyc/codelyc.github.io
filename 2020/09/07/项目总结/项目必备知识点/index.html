异构网络必备知识点

一些细碎知识点详解

虚拟地址

对物理地址进行映射的地址

虚拟内存

呈现出要比实际拥有的内存大得多的内存量。具体操作类似：你不必非常长的轨道就能让一列火车从上海开到北京。你只需要足够长的铁轨（比如说3公里）就能完成这个任务。采取的方法是把后面的铁轨即时铺到火车的前面，只要你的操作足够快并能满足需求，列车就能象在一条完整的轨道上运行。

内核空间和用户空间

操作系统将虚拟空间划分为两部分，一部分是内核空间，一部分是用户空间。将最高的1G字节（从虚拟地址0xC0000000到0xFFFFFFFF）供内核使用，称为内核空间，而较低的3G字节（从虚拟地址0x00000000到0xBFFFFFFF），供各个进程使用，称为用户空间。在Linux系统中，进程的用户空间是独立的，而内核空间是共有的，进程切换时，用户空间切换，内核空间不变。

内核空间

它独立于普通的应用程序，可以访问受保护的内存空间，也有访问底层硬件设备的所有权限。

用户空间

为了保证内核的安全，用户空间不能不能直接操作内核

内核态和用户态

当进程运行在内核空间时就处于内核态，而进程运行在用户空间时则处于用户态。在内核态下，进程运行在内核地址空间中，此时 CPU 可以执行任何指令。运行的代码也不受任何的限制，可以自由地访问任何有效地址，也可以直接进行端口的访问。而用户态受到诸多限制

如何从用户空间进入内核空间

有三种方式：系统调用、软中断和硬件中断。其实就是通过一个特殊的指令让进程从用户态进入到内核态(到了内核空间)。

【总结】：用户空间中的代码只有通过内核暴露的系统调用接口(System Call Interface)才能使用到系统中的硬件资源。

iptables

访问控制

我们自己写代码制定一系列的规则，而内核空间可以将这些规则组成一个列表，实现绝对详细的访问控制功能。这些规则都工作在用户空间中，但可以让内核空间的netfilter读取，并将这些规则放在tcp/ip的协议栈进过的地方

netfilter规定的规则链

  1. PREROUTING (路由前)

  2. INPUT (数据包流入口)

  3. FORWARD (转发管卡)

  4. OUTPUT(数据包出口)

  5. POSTROUTING（路由后）

​ 这是NetFilter规定的五个规则链，任何一个数据包，只要经过本机，必将经过这五个链中的其中一个链。
