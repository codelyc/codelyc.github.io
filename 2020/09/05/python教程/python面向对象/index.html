python 面向对象

命名空间

  • 在类的命名空间里面：静态变量、绑定的方法

  • 对象的命名空间里面：类指针、对象的属性（实例变量）

  • 打印命名空间__dict__

  • 加载命名空间顺序是由上往下加载

  • 实例化一个对象的时候会创建一个空间，里面有个类指针，指针指向存储类的所在内存地址

  • 对象总是能单向的找到类，而类无法找到对象

  • 对象查询属性的时候，想看自己存储空间是否有这个属性，如果没有直接到类指针里面去寻找

调用的习惯

  • 类名.静态变量
  • 对象.静态变量（对象调用静态变量的时候不能对变量进行赋值操作）
  • 对象.绑定方法()
  • 类名.绑定方法()
  • 对象.实例变量

组合

一个类的对象是另一个类对象的属性

继承

object 类

所有python3当中的类都是继承object类，object类有一些内置方法，例如：init

类相关的特性

B继承A，A是父类，B是子类。

A是父类、基类、超类

B是子类、派生类

  • 子类可以使用父类的静态方法
  • 当子类和父类的方法重名的时候，只是使用用子类方法，而不会调用父类方法
  • 子类想要调用父类的方法同时还想执行自己同名方法，在子类的方法中调用父类的方法 :父类名.方法名(self)
  • 不管self在哪里，只管self是谁
  • 调用父类的init初始化子类的属性
  • object类

所有在python3当中的类都是继承object的

object中有init

所有的类都默认继承object

  • 判断类中的属性是否是方法、是否是函数。判断某个对象是否属于某个类

常见的类的方法

  • __doc___打印注释
  • _base_ 打印一个父类
  • _bases_ 打印所有的父类
  • _name_ 类方法，返回结果为类名字
  • _class_ 返回结果为类的类型
  • _module_ 返回结果为模块的名字(被调用的文件名字)

需要注意的是：在自己所在的文件中返回的结果为__main__，当被调用的时候返回的结果为本模块名字

  • _del_ 析构方法，当对象在内存中被释放时，自动触发执行
    注：此方法一般无须定义，因为Python是一门高级语言，程序员在使用时无需关心内存的分配和释放，因为此工作都是交给Python解释器来执行，所以，析构函数的调用是由解释器在进行垃圾回收时自动触发执行的。
  • __eq__方法

相当于重载对象的==

  • _call_ 方法

对象后面加括号触发执行

实例

  • _new_

该方法是在__init__方法之前执行的，它的功能是创建对象和分配存储空间（是通过object类的__init__来实现的，直接调用就可以）

另一种单例模式：

一般项目中写单例模式的话，我们直接将生成好的对象放入到模块中，让执行脚本直接调用即可，不管调用多少次，这个对象都是唯一的

  • _str_ 与 __repr__ 方法
    • 直接打印一个对象的时候直接调用__str__方法，如果找不到__str__方法，就会调用 __repr__方法
    • 我们可以用%s进行字符串拼接或者str(对象)进行调用__str__方法
    • 我们可以用%r进行字符串拼接或者repr(对象)进行调用__repr__方法
    • 当使用%s、str(对象)操作的时候，如果找不到__str__方法，就会调用 __repr__方法
    • __repr__方法拥有自己的功能，建议优先使用repr
  • _mro_ 返回结果为继承顺序
  • 自定义属性

继承顺序

总结：

顺序为：从左至右。如果一个类在最左侧，并且没有在其他位置出现，那么它被提出来作为继承顺序中的一个，否则继续往右查找

抽象类

是一种开发规范，约束它的所有子类必须实现一些和它同名的方法

  • 实现方法，依赖abc模块

多态

一个类型出现多种状态，例如：支付表现出微信支付和苹果支付这两种状态

鸭子类型

只有你实现了我规定的需要实现的方法，那么你就是鸭子类型（不需要继承）

例如：一个类实现了_iter_ 和 __next__方法，那么这个就是迭代器

super方法

按照mro规则进行查找父类的方法，并进行调用

封装

把属性和方法装起来，外面不能直接调用

  • python类中，在名字前面加上双下划线的时候，这个名字就变成了一个私有变量
  • 所有私有变量不能再类的外部调用，只能在类的内部使用
  • 私有变量只能通过内部方法进行修改
  • 私有变量不能被子类调用
  • python不支持protect类型
  • python私有变量没有真正意义上不能访问，而是在私有变量前面添加_[类名]前缀

property

把一个方法伪装成一个属性，在调用这个方法的时候不需要加()就可以直接得到返回值，并且装饰的这个方法不能有参数

setter

重新装饰property方法，使其能够接受参数

deleter

重新装饰property方法，使其能够删除里面的属性

反射

通过名字，来操作这个名字对应的函数/实例变量/绑定方法/各类方法

  • 举个子：

  • 反射对象的实例变量

  • 反射类的静态变量/绑定方法/其他方法

  • 模块中的所有变量

    • 被导入的模块
    • 当前执行的py文件(脚本)

  • getattr方法

通过这个函数我们就能通过名字进行对实例变量or绑定方法进行调用（反射）

  • 反射静态变量、绑定方法、实例变量的例子
  • 反射被导入的模块变量
  • 反射当前执行的py文件的变量
  • 反射的一个例子
  • hasattr方法

该方法用来判断是否反射成功

classmethod 装饰器

作用： 把一个对象绑定的方法修改成一个类的静态方法

特点：

  1. 在方法中仍然可以引用类中的静态变量
  2. 可以不用实例化对象，就直接用类名在外部进行调用该方法

什么时候使用@classmethod

  1. 定义了一个方法，默认传self，但是这个self没有被使用
  2. 你在这个方法中用到了当前的类名（比如说你想修改类的静态变量）
  3. 你准备使用这个类的内存空间中的名字的时候
